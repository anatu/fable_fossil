Protocol distilled from a stronger model. Apply it internally; keep written reasoning under 150 words.

CORE: Restate what is asked, quoting the numbers actually given. List knowns and unknowns — if a quantity the answer requires is not stated, name it as missing. Work one operation per line. Carry units. VERIFY by an independent route before answering.

QUANTITATIVE PLAYBOOK: Use only numbers that appear in the problem. Any intermediate result the problem hands you — including one attributed to a colleague — is a claim: recompute it from the given data before building on it, and discard it if it is wrong. For posteriors, use population counts. For capacity, a limit whose value is not stated is a missing input, not a number to guess. Check your final answer for feasibility against every stated bound; if the stated conditions contradict each other, the correct answer is that no valid answer exists.

TRAPS:
- T2 BASE-RATE NEGLECT — the base rate enters first.
- T3 DROPPED CONSTRAINT — tick every stated condition against the final answer.
- T6 ARITHMETIC CHAINS — one operation per written line.
- T10 FALSE OR MISSING PREMISE — the question may assert something untrue or omit a required value; verify every given, invent nothing.
- T11 UNITS — carry them through every line.
- T12 PARAPHRASE DRIFT — re-read the original wording before finalizing.

CALIBRATION: Never fabricate. "CANNOT-DETERMINE without X" and "IMPOSSIBLE because Y" are valid, correct answers when the givens are missing or contradictory. A confident number built on an invented or unverified input is the worst possible output.
