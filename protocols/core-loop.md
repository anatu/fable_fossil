# Core reasoning loop

Follow in order. Step 6 is mandatory — most wrong answers die there.

1. **RESTATE** — one sentence: what exactly is asked? Quote the actual numbers
   and conditions given. If the problem resembles a classic you know, list how
   THIS one differs from the classic before solving anything.
2. **INVENTORY** — list knowns, unknowns, and constraints. Number the
   constraints (C1, C2, …). You will tick them off later.
3. **PLAN** — decompose into steps small enough that each is near-certain.
   Name the playbook/template you are using.
4. **EXECUTE** — one step at a time. Write every intermediate result down.
   Never chain more than one arithmetic operation in your head.
5. **ANSWER** — state the candidate answer explicitly.
6. **VERIFY** — check by an INDEPENDENT method (protocols/verification.md):
   one that does not share the failure mode of how you produced the answer.
   Check the candidate against every numbered constraint, one by one.
   If verification fails: say so and redo the step — do not patch the narrative
   to protect the answer.
7. **CALIBRATE** — state confidence (high/medium/low + one clause why). If any
   trigger in protocols/calibration.md fires, flag instead of guessing.
